# CLAUDE.md

This file provides guidance to AI assistants working on the TanStack Chat Template codebase.

## Project Overview

A full-stack AI chat application built with TanStack Start (React 19) and the Anthropic Claude API. It features streaming responses, markdown rendering, optional Convex database persistence, and deploys to Netlify.

## Commands

- `npm run dev` - Start development server (vinxi dev)
- `npm run build` - Production build (vinxi build)
- `npm run serve` - Preview production build (vite preview)
- `npm run start` - Start production server (vinxi start)

There is no test runner, linter, or formatter configured in this project.

## Architecture

### Tech Stack

- **Framework**: TanStack React Start 1.x on Vite 6 + Vinxi
- **UI**: React 19, Tailwind CSS 4
- **Routing**: TanStack Router (file-based, auto-generates `src/routeTree.gen.ts`)
- **State**: TanStack Store (`src/store/`) with local-first pattern + optional Convex sync
- **AI**: Anthropic SDK (`@anthropic-ai/sdk`) - server-side only via `createServerFn`
- **Database**: Convex (optional, enabled when `VITE_CONVEX_URL` is set)
- **Deployment**: Netlify (configured in `app.config.ts` with `preset: 'netlify'`)
- **Error Tracking**: Sentry (optional, enabled when `SENTRY_AUTH_TOKEN` is set)

### Directory Structure


### Key Patterns

**Server Functions**: AI calls use `createServerFn` from TanStack Start (`src/utils/ai.ts`). The `genAIResponse` function runs server-side, streams Claude responses as NDJSON, and keeps the API key off the client.

**State Management**: Two hooks provide state access:
- `useAppState()` - UI state: prompts, loading, banner visibility
- `useConversations()` - Conversation CRUD with local-first + optional Convex sync

The local-first pattern means all actions update TanStack Store immediately for instant UI feedback, then asynchronously sync to Convex if available. Convex availability is determined at module load time via `import.meta.env.VITE_CONVEX_URL`.

**Streaming**: AI responses stream character-by-character with adaptive rendering speed (faster for code blocks, smoother for regular text). The server sends NDJSON; the client parses line-by-line and queues text chunks for animated display.

**Routing**: File-based routing under `src/routes/`. The app directory is set to `src` in `app.config.ts`. `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` - never edit it. Route components export `Route` using `createFileRoute` or `createRootRoute`.

## Environment Variables

| Variable | Required | Scope | Purpose |
|----------|----------|-------|---------|
| `ANTHROPIC_API_KEY` | Yes | Server only | Claude API key (no `VITE_` prefix to keep server-side) |
| `VITE_CONVEX_URL` | No | Client + Server | Enables Convex database persistence |
| `VITE_SENTRY_DSN` | No | Client | Sentry error tracking DSN |
| `SENTRY_AUTH_TOKEN` | No | Build time | Sentry source map uploads |

Copy `.env.example` to `.env` for local development. Never commit `.env`.

## TypeScript Configuration

- Target: ES2022, strict mode enabled
- `noUnusedLocals` and `noUnusedParameters` are enforced
- `verbatimModuleSyntax: true` - use `import type` for type-only imports
- Module resolution: `bundler` mode (Vite handles resolution)
- No emit - TypeScript is only used for type checking; Vite handles transpilation

## Code Conventions

- Components are in `src/components/` with a barrel export in `index.ts`
- Store actions are pure functions that call `store.setState()` with immutable updates
- Selectors are plain functions taking `State` and returning derived values
- Server-only code uses `createServerFn` - these functions cannot access browser APIs
- Client-side env vars must use the `VITE_` prefix; server-only vars must not
- The AI model is `claude-sonnet-4-5-20250929` (configured in `src/utils/ai.ts:112`)
- Icons come from `lucide-react`
- Markdown rendering uses `react-markdown` with `rehype-highlight`, `rehype-raw`, and `rehype-sanitize`

## Files to Never Edit Manually

- `src/routeTree.gen.ts` - Auto-generated by TanStack Router plugin
- `convex/_generated/` - Auto-generated by Convex
